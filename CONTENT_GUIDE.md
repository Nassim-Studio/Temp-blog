# Content Guide for GitHub Pages Blog (Native Jekyll)

Follow these steps to add a new post to your blog directly from GitHub:

1. Go to your GitHub repository website.
2. Navigate to the `/_posts/` directory.
3. Click **Add File** -> **Create new file**.
4. Name your file strictly using the format: `YYYY-MM-DD-your-title.md` (e.g., `2024-03-17-my-first-post.md`).
5. Copy and paste the template below into the new file, then write your content below it:

```markdown
---
layout: post
title: "Your Title Here"
---

Your content goes here...
```

6. Click **Commit changes** to save and publish your post. 
7. GitHub Pages will natively build your site within a few seconds!

### Pages

To edit standalone pages such as About Us or Partners:
1. Navigate to the root directory `about.md` or `partners.md`.
2. Click the edit (pencil) icon.
3. Edit the Markdown text below the frontmatter (`---` lines).
4. Commit your changes.
